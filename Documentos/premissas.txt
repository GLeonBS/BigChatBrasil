Sistema de mensagens: BCB – Big Chat Brasil

Um sistema de envio de mensagens
Quando o cliente solicitar o de envio da mensagem - Deve ser verificado o tipo de plano de pagamento
Caso for pré-pago, verificar os créditos do cliente para envio que custam 0,25 cada
Caso for pós-pago, registrar o consumo na conta até o limite máximo autorizado

Um cliente terá, Id, Nome, e-mail, telefone, CPF responsável, CNPJ, Nome da empresa, e CONTA(VO):(Plano, créditos, limite)
Um destinatario terá, Id, Nome, telefone

Mensagem - id, número de telefone, é whatsapp?, texto, chat, data/hora do envio
MensagemHistórico - Cliente, Plano, Consumido, DataHota, Mensagem
CHAT - id, remetente, destinatários ? podem ser apenas números, mensagens
Cada cliente precisa ser préviamente cadastrado

Um cliente enviará para vários usuários finais?
Acredito que sim, com base na frase "Precisamos que seja possível via web ou mobile que os clientes enviem mensagens para seus usuários finais"

É via websocket?
Acredito que não, considerando que a mensagem pode ser via WhatsApp, não é um chat em tempo real, e sim uma mensagem que será enviada para os usuários finais

Os usuários finais são cadastrados?
Acredito que sim, pois precisamos saber quem são os destinatários das mensagens, preciso mesmo? E se eu só informar os números de telefones?

O usuário final pode responder a mensagem?
Acredito que não, pois não foi mencionado que o usuário final pode responder a mensagem

O financeiro poderá:
- Incluir créditos para um cliente
- Consultar saldo de um cliente
- Alterar o limite de um cliente
- Alterar o plano de um cliente
- Ver dados de um cliente

Domínios Principais:
Contexto de cadastro:
    - Cliente
    - Mensagem
    - Usuário final(destinatário)
    - Chat
